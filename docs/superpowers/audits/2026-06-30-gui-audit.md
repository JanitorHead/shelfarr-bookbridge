# shelfarr-bookbridge — GUI Audit

**App:** Go (stdlib `net/http` + `html/template` + `go:embed`), *arr-style self-hosted, deployed on Unraid behind a reverse proxy.
**Scope:** The web GUI only (dashboard, settings, shelves, queue, review, login).
**Date:** 2026-06-30
**Method:** 10-agent audit workflow — 3 mappers (handlers / templates+CSS / data-flow) → 6 parallel dimension auditors → 1 synthesis lead. 46 findings, every one anchored to `file:line`.

---

## Executive summary

The GUI is accurate to the owner's complaints. It is a single ~11-line dark stylesheet over plain server-rendered HTML with **zero JavaScript**, and it fails the two questions an *arr user opens the app to ask: *"is a sync running?"* and *"what happened last run?"*.

The backend already tracks everything needed — a single-flight run lock with timestamp (`run_state.running/started_at`), an atomic Acquire/Release, and an 11-counter `Report` — but almost none of it reaches the screen. `run_state` has **no store reader at all**, and the only run surface (the `LAST_RUN` setting) is written **exclusively by the manual web button**, so hourly scheduled syncs and CLI runs leave **zero trace**. "Can't force the sync" is real in effect: the button works but runs **synchronously inside the HTTP request**, freezing the tab for minutes and timing out behind the Unraid reverse proxy, with no spinner and no double-submit protection.

Goodreads RSS-vs-cookie is chosen **invisibly** by cookie presence; the one field that looks like a mode picker (`GOODREADS_VISIBILITY`) is a **dead no-op** wired to nothing; the 100-item RSS truncation is a server-log-only warning. The "no dropdowns" complaint is exact — a `Kind` discriminator exists in Go and the CSS already styles `<select>`, but the render struct drops `Kind`, so **all 14 settings (seven enums, three booleans, raw 5-field cron) render as identical free-text boxes with no validation**, persisting typos that can crash the daemon. Compounding everything, a strict **Content-Security-Policy silently strips every inline style**, so the few intended green/red/amber cues never render. Finally, **Hardcover is unsupported end-to-end** — though the engine is already source-agnostic, so the fix is contained.

Every fix lands inside the stated stack (`net/http` + `html/template` + `go:embed` + one CSS file + a tiny vendored `app.js`). **No SPA framework required.**

## Top problems

1. **Scheduled & CLI syncs leave ZERO trace in the GUI** — `LAST_RUN` is written only by the manual web button, so after a night of hourly cron syncs the dashboard still shows the last time the owner personally clicked the button. (`cmd/bookbridge/main.go:189-196` vs `internal/web/handlers.go:193`)
2. **No "is a sync running now" indicator** — `run_state.running/started_at` has no store reader (`internal/store/lock.go`), so the dashboard is blind to in-flight runs.
3. **CSP silently blocks every inline style** (`server.go:113`, `default-src 'self'`, no `style-src`) — hidden root cause of "nothing to look at, no sense of state". Every button blue, every banner green.
4. **"Sync now" runs synchronously inside the HTTP request** (`handlers.go:180`) — the tab hangs for minutes and times out behind the reverse proxy while the engine keeps running detached, so the button looks dead.
5. **Every Settings field renders as free-text** despite a `Kind` discriminator in Go and `<select>` styling in CSS — 7 enums, 3 booleans, raw cron typed by hand with no validation; a typo'd enum or cron is persisted silently and can crash the daemon.
6. **Goodreads RSS-vs-cookie chosen invisibly** by cookie presence; `GOODREADS_VISIBILITY` is a dead decoy wired to nothing; the 100-item RSS truncation is invisible; no test-connection.
7. **Hardcover can't run at all** — source hardcoded to `goodreads.NewSource` in two places (`main.go:76,136`), no `Source` config field — though the engine is already source-agnostic, so the fix is contained.

## Implementation gotchas (from the audit)

- **Checkbox/select POST gotcha:** the Settings POST loop only saves *non-empty* values (`handlers.go:61`), so an unchecked checkbox submits nothing and can never turn a setting **OFF** — checkbox/select fields must always write a canonical value (mirror the shelves `Enabled` handling at `pages.go:48`).
- **Keep CSP strict.** Every "add interactivity" item (R4, R14, R16) uses a single vendored `/static/app.js` added to the `go:embed` set and loaded from `base.html` — `default-src 'self'` already permits it. No inline scripts, no CDN. Provide a `<meta refresh>` no-JS fallback for status.
- **Schedule live-reload caveat:** `cfg` is captured at daemon startup (`main.go:183`), so a GUI schedule change (R6) won't take effect until a restart. Surface "restart required" in the UI until a config-reload path exists.
- **Hardcover needs no engine changes:** `engine.New` takes any `sources.Source`; results flow into `store.Diff` keyed by `(source, external_id)`; resolver/reconcile/review are all source-generic. A Hardcover source emitting `Source="hardcover"` coexists with Goodreads rows.
